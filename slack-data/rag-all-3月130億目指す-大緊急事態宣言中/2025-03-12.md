  eventDate,
  entry_error_case

  eventDate,

<@U079ZUQ61BR> cc <@U03A90VQF17>
独立EFにて
・PDT1で無効(遵法外)の方がPDT1に新規エントリーしたuu（エントリー成功・失敗の両方カウント）
・PDT1で無効(判定N)の方がPDT1に新規エントリーしたuu（エントリー成功・失敗の両方カウント）
って、分析する方法ありますでしょうか？
`dharma-dwh-rag.wc_resource_new.t_jobseeker`で無効(遵法外)か無効(判定N)かが取れるのはわかっています。
ただ、パーティションテーブルじゃないから、「PDT1に新規エントリーした直前までは、無効(遵法外)だったのか、無効(判定N)だったのか」をどう取ればいいのかで悩んでました。

```SELECT
  case
    when jobseeker_stat_code = "10" then "01_有効"
    when jobseeker_stat_code = "11" then "02_無効(遵法外)"
    when jobseeker_stat_code = "12" then "03_無効(判定N)"
    when jobseeker_stat_code = "20" then "04_無効(決定計上)"
    when jobseeker_stat_code = "99" then "05_無効(活動停止)"
    when jobseeker_stat_code = "00" then "06_無効(未判定)"
  end as js_stat_name,
  count(distinct jobseeker_no||jobseeker_branch_no) as js_cnt,
FROM
  `dharma-dwh-rag.wc_resource_new.t_jobseeker`
GROUP BY ALL
ORDER BY 1```
