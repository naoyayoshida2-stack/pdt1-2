## 15:50 U027HAZE3CH
<@U050FKVT5L1> <!subteam^S06T2SCEX41>
レジュメPDF取得処理について
設計進めてみてちょっと仕様を変えておりますのでこちらで一報入れさせていただきます！

BE側のレジュメ取得APIは共通で開発予定でしたが、
CALIBからの呼び出しと、外のAPIからで呼び出されるモジュールの場所が変わるため別モジュールで実装するようにします。
そのため、CALIBボタンからと応募API経由で下記のようにアクションを変更するようにして進めております。
（こっちの方がUX的にも使いやすいと想定しております）

■CALIBのボタンから実行した場合
　・レジュメの更新日時に差分があれば
　　└最新レジュメを取得する
　　　└成功したら成功メッセージを表示する
　　　└失敗したら失敗メッセージを表示する
　・レジュメの更新日時に差分がない
　　└最新になっている旨を伝えるメッセージを表示する
　※CALIBボタン経由ではコミュは刺さらない
　
■応募ボタンが押下された場合
　・レジュメの更新日時に差分があれば
　　└最新レジュメを取得する
　　　└正常したら、成功の通知コミュをCAに刺す
　　　└失敗したら、失敗の通知コミュをCAに刺す
　・レジュメの更新日時に差分がなければなにもしない

<https://one-recruit.slack.com/archives/C06DERWPZ9D/p1712110097075419?thread_ts=1711687023.079529&cid=C06DERWPZ9D>

## 16:24 U01SY68B10V
<@U01U4HJ7D42>
汚いけど要諦Docで使ったグラフの元となってるクエリこちら、ここに送付手段系とかもくっつけたりしながら見立て立ててもらえるとありがたし！
```with a as(
  select
    jobseeker_id,
    jobseeker_branch_id,
    sum(cast(ca_recommend_flag as int64)) as apply_cnt,
    sum(cast(doc_pass_flag as int64)) as ok_cnt    
  from
  dharma-dwh-rag.datamart.t_rag_application_funnel
  where
  apply_date >= '2023-04-01'
  group by
  jobseeker_id,
  jobseeker_branch_id
),

b as (
  select
    *
  from
  dharma-dwh-rag.datamart.t_rag_jobseeker_all
  where
  date(entry_timestamp) between '2023-04-01' and '2023-09-30'
),

c as (
  select
    a.*,
    b.entry_timestamp,
    b.ca_interview_date,
    b.acceptance_flag
  from
  b
  left outer join
  a
  on
  b.jobseeker_id = a.jobseeker_id
  and b.jobseeker_branch_id = a.jobseeker_branch_id
),

d as (
  select
    jobseeker_id,
    jobseeker_branch_id,
    concat(jobseeker_id,jobseeker_branch_id) as jobseeker_no,
    case when ca_interview_date is not null then '1_面談済み' else '2_未面談' end as mendan_f,
    apply_cnt,
    ok_cnt
  from
  c
),

e as (
  select
    case
      when apply_cnt = 0 then '00_0'
      when apply_cnt <= 5 then '01_1-5'
      when apply_cnt <= 10 then '02_6-10'
      when apply_cnt <= 15 then '03_11-15'
      when apply_cnt <= 20 then '04_16-20'
      when apply_cnt <= 25 then '05_21-25'
      when apply_cnt <= 30 then '06_26-30'
      when apply_cnt > 30 then '07_31-'
    else '00_0' end as apply_bin,
    mendan_f,
    sum(1) as uu,
    sum(case when ok_cnt >= 1 then 1 else 0 end) as ok_uu,
    sum(case when ok_cnt >= 1 then 1 else 0 end) / sum(1) as entry_ok_rate
  from
  d
  group by
  apply_bin,
  mendan_f
)

select 
  *
from
e
order by
apply_bin,
mendan_f```
